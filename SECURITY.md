# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities privately. Use this repository's private GitHub vulnerability reporting flow if it is enabled. Do not open public issues or pull requests for active vulnerabilities until a maintainer has reviewed the report.

When reporting a vulnerability:

- Include a clear description of the issue and affected area.
- Include reproduction steps only as needed for maintainers to validate the report.
- Include impact, prerequisites, and any known mitigations.

Do not publish any of the following in public issues, pull requests, discussions, or comments:

- Credentials or passwords
- Session cookies
- API keys or tokens
- Personal data
- Full exploit details or weaponized proof-of-concept material

If sensitive data was exposed accidentally, rotate or revoke it before sharing follow-up context.

## Supported Versions

Recipe Clipper `1.0.0` is the initial public release baseline.

The following versions are currently supported:

- `1.x` releases
- The latest code on the `main` branch

The following are not currently supported:

- Older snapshots, forks, or unpublished local modifications
- Historical commits not represented by the current `main` branch state

## Response Expectations

Maintainers will review private reports and determine remediation and disclosure timing. Please avoid public disclosure until a fix or mitigation plan is ready.
